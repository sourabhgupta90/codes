Integrating TopCoder Arena in Ubuntu Menu

fushar    Tips     topcoder, ubuntu

Are you a regular competitor of TopCoder SRMs? If so, then probably you have been launching the Arena by (double) clicking a sacred file named ContestAppletProd.jnlp somewhere in your computer.

I felt that looking for that file before every SRM is quite tedious, and putting the file in my desktop is not an elegant way. So, I added a launcher to TopCoder Arena in Ubuntu GNOME menu. It’s very handy, so I would like to share how to do that. In this tutorial, I am using Ubuntu 11.04 Natty Narwhal.

  • Download ContestAppletProd.jnlp from TopCoder website, by clicking the O(n) icon in the upper left corner. Save it, for example, in /home/fushar/TC.
  • Install Sun Java 6 JRE. There are many ways to do this, consult Google for references. For example, you can read this nice tutorial.

You must have done the two steps before reading this post, eh? Now let’s add it to Ubuntu menu.

  • Right-click GNOME menu, and click Edit Menus. A new window will appear.
  • Click Internet, then Sun Java 6 Web Start, then Properties. You will see something like this:

    Copy the entire text in Command field. After that, click Close.
  • Still in the same window. Click Programming, and then New Item. Enter these details:
    • Type: Application
    • Name: TopCoder Arena
    • Comment: Launch TopCoder Arena (or anything you want)
  • Paste the text into Command field. Then, replace %u with the path to ContestAppletProd.jnlp file, enclosed in double quotes. In my case, the text would be:
    /usr/lib/jvm/java-6-sun-1.6.0.26/bin/javaws “/home/fushar/TC/ContestAppletProd.jnlp”
  • (Optional) If you want an icon for the menu, then download this image, save it somewhere, click the icon button in the left, and then select the image. The image is basically a JPG-converted image from TopCoder website’s favicon.
  • Click Close.

Done! You should be able to see the new menu in Applications -> Programming. From now on, you can forget where you place the ContestAppletProd.jnlp file, and run the Arena directly from Ubuntu menu instead.

Bonus tip: If you add -Xnosplash before the ContestAppletProd.jnlp path, like this:
/usr/lib/jvm/java-6-sun-1.6.0.26/bin/javaws -Xnosplash “/home/fushar/TC/ContestAppletProd.jnlp”
then the Java splash logo will not appear before the Arena loads.

Try this! I’d love to hear how this tutorial helps you guys
